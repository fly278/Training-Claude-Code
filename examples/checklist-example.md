# 执行前检查清单 — 详细示例

> 这是 SELF_CHECK.md 中"计划先行协议"的完整示例，展示如何应用自查流程。

---

## 示例任务: "给 connector 添加附件发送功能"

### Step 1: 需求澄清
- [ ] 附件来源是什么？本地文件路径 / 用户上传 / 生成的 PDF？
- [ ] 附件大小限制？单个 / 总计？
- [ ] 支持哪些格式？任意 / 仅特定类型？
- [ ] 失败时怎么处理？跳过附件 / 整封邮件失败？

### Step 2: 影响范围评估
- [ ] 需要修改哪些文件？
- [ ] 依赖关系检查 → 查 DEPENDENCY_MAP.md
- [ ] 是否需要新依赖？
- [ ] 是否需要新的环境变量？

### Step 3: 技术方案评估
- [ ] 方案 A: 修改现有命令，增加参数
- [ ] 方案 B: 新增独立命令
- [ ] 选择: 方案 A — 保持接口简洁，向后兼容

### Step 4: 安全检查
- [ ] 文件路径需要校验（防路径遍历）
- [ ] 大小限制（常见服务器限制）
- [ ] 敏感文件过滤（.env, *.key, *credentials*）

### Step 5: 测试计划
- [ ] 健康检查: 改动前后各一次
- [ ] 基础功能: 回归测试
- [ ] 新功能: 正常路径验证
- [ ] 边界 case: 不存在的文件 / 空文件 / 超大文件
- [ ] 安全 case: 路径遍历尝试

### Step 6: 回滚计划
- [ ] 能否快速回退？
- [ ] 是否需要备份？
